# 山海经原典地图研究台 · 桌面版 v004

桌面版 v004 继续保留 v003 的逐帧地图拖动、鼠标右键返回与签名更新系统，本轮重点调整地块博物志“简述”页，并增加 Markdown 批量选择能力。

## 本版功能

- “简述”模式移除自动摘要，改为对象级图鉴陈列。
- 只显示当前地块实际存在的分类；没有对象的分类不会占用版面。
- 每个对象拥有独立图片区。尚未准备图片时显示统一“待补图”占位，不影响对象资料导入。
- 图片加载失败会自动退回占位状态，避免出现破图。
- 对象编辑与 Markdown 均支持可选“图片”字段。
- Markdown 工作台可选择单个文件，也可一次选择多个文件统一分析。
- 批量结果显示来源文件、识别条目、警告和错误；错误条目不会写入地图。
- 内置 v075 母表：395 个对象、2432 条原文记录。


## v0.4.3 九段式 Markdown 博物志导入

- 支持直接批量导入 `01.区域定位` 至 `09.详细描述` 的九段式 Markdown。
- 每个文件通过文件名匹配客户端中的既有对象，例如 `九疑山.md` 对应对象“九疑山”。
- 导入只更新该对象的博物志资料，不改变对象坐标、几何类型和锁定状态。
- 第06节分类条目会尝试绑定同名地图对象；无法唯一匹配时保留为文字卡片，不丢失内容。
- 原有 `### 对象：` 与 `### 地块档案：` 两种导入格式继续可用。
- 精确对象预览窗内部可滚动时优先滚动列表；到达顶部或底部后恢复地图缩放。

九段式文件批量导入方式：进入“导入 Markdown”工作台，点击“批量选择 .md”，一次选中全部文件，核对结果后点击“添加到本地地图”。

## 当前图片边界

v004 已完成对象级图片区和图片字段，但尚未加入“从任意本地文件夹复制图片到程序素材库”的批量图片管理器。当前图片字段可以留空。后续准备好插画后，再通过专门的对象图片导入功能完成文件复制、匹配与绑定，避免本地路径移动后失效。

## Markdown 批量选择

在“导入 Markdown”工作台中可使用：

```text
选择单个 .md
批量选择 .md
```

批量限制：最多100个文件；单个文件不超过5MB；总量不超过25MB。每个对象可选填写：

```markdown
* 图片：
```

留空时对象仍会正常导入，并在简述页显示“待补图”。

## 推荐发布流程

1. 在本地测试并确认 `npm run verify` 通过。
2. 使用 GitHub Desktop 提交并推送到 `main`。
3. 等待 `Build Windows Desktop` 变绿。
4. 在 GitHub Actions 手动运行 `Publish Windows Desktop Update`。
5. 在 Releases 确认 `latest.json`、安装程序与 `.sig` 已生成。

根目录的 `发布新版本.cmd` 仅作为可选辅助工具，不是发布所必需的步骤。

## 本地测试

```powershell
npm ci --registry=https://registry.npmjs.org/
npm run verify
npm run build
npm run desktop:dev
```

正式更新包由 GitHub Actions 使用既有签名密钥生成。后续版本必须继续使用与 v002 相同的签名私钥，否则旧客户端无法验证更新。

## v0.4.1 更新通道

## v0.4.2 精确点与数据发布

- 简述博物志分类列表超过可视高度后在分类内部滚动。
- 精细地图中的单对象点直接打开精确点博物志；同坐标多对象先展开列表，再选择对象。
- “完成本轮编辑”通过本地 Git 仓库写入 `submissions/pending/*.shjpatch` 并推送 `main`。程序不会保存 GitHub Token；需本机已安装并登录 GitHub Desktop，或已配置可用的 Git。
- 首次自动上传若未找到仓库，填写仓库根目录，例如 `F:\SHmap\SHmap`。上传失败不会清空本轮编辑。
