# CHANGELOG

## Desktop v004.2（整合版）

- 简述博物志的分类栏增加独立纵向滚动区，单个分类对象过多时不再把整页无限拉长。
- 420%以上精细地图中的精确点支持打开博物志；同坐标多对象先选择具体对象，再打开精确点档案。
- 精确点博物志仅展示同一精确坐标的对象，并保留简述、完整、定位和复制摘要功能。
- “完成本轮编辑”会先保存桌面数据库，再生成 `.shjpatch`、提交到 `submissions/pending` 并推送 GitHub。
- 自动上传不内置密钥，复用本机 GitHub Desktop / Git 登录；网络失败时保留本轮更改并提供重试与本地导出。
- 上传前检查 main 分支与远端同步状态，避免顺带推送未确认的代码提交。
- 地图滚轮缩放与程序内定位动画改为逐帧真实重绘，移除临时缩放图层回基准导致的视觉回弹。
- 缩放手感改为按滚轮增量连续计算，鼠标指针下的地图位置保持稳定。
- 进入／退出区域概览和10里精细地图时取消淡入动画，模式切换不再出现二次缩放错觉。
- 精细地图悬停对象点时，为同一精确坐标下的每个对象分别弹出独立预览小窗。
- 单击多对象点可固定／取消固定预览组；单击任一小窗打开对应精确点博物志，双击可编辑该对象。
- 预览组靠近窗口边缘时自动向左或向上展开，并在对象较多时内部滚动。

## Desktop v004.1

- 增加仓库直连、jsDelivr CDN 与 GitHub Releases 三条更新检查线路。
- 每条检查线路自动重试，连接重置或超时后自动切换下一线路。
- 更新包下载增加最多3次自动重连，避免短暂断网直接导致安装失败。
- 发布成功后自动同步签名 `latest.json` 到仓库稳定更新源。
- 更新窗口显示当前使用线路、重试状态与完整失败原因。

## Desktop v004

- 仅重做地块博物志“简述”模式，完整模式与原文证据区保持原有逻辑。
- 移除简述页自动摘要和固定六类概览，改为只显示有对象的分类。
- 每个对象卡片增加独立图片区、名称、关系、核心特征、功效与证据摘要。
- 没有图片或图片加载失败时自动显示统一“待补图”占位，不出现破图标记。
- 对象编辑与 Markdown 导入增加可选图片字段，为后续图片素材导入预留绑定接口。
- Markdown 工作台增加单文件与批量文件选择；批量导入显示文件清单与条目来源。
- 批量选择限制为最多100个文件、单文件5MB、总量25MB，避免一次载入过量文本。

## Desktop v003

- 地图拖动改为实时视觉跟随；画布、画笔轨迹与地块标签同步移动。
- 拖动中使用 GPU `translate3d`，按距离进行轻量重基准，松开后再完整重绘和保存。
- 增加鼠标右键返回，支持关闭弹窗、工作台、地块翻面与空间聚焦。
- 修复更新检查完成后进度文字仍停留在“正在检查”的状态。
- 增加根目录一键发布脚本，自动同步版本、校验、构建、提交、打标签并推送。
- 正式发布工作流改为监听 `desktop-v*` 标签并自动创建 GitHub Release。
- 构建产物上传动作升级，移除旧 Node 运行时警告。
