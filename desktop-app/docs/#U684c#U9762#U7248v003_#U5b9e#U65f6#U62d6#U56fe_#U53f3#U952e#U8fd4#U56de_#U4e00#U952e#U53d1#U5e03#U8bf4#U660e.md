# 桌面版 v003 更新说明

## 一、地图实时拖动

v003 将地图拖动拆成两个阶段：

1. 拖动中：地图画布、画笔轨迹和地块标签使用同一组 `translate3d` 位移，画面立即跟随鼠标。
2. 拖动结束：提交最终相机坐标，进行一次完整重绘并保存视口位置。

长距离拖动时会按距离重新绘制当前视口，避免画布边缘出现大面积空白。拖动中的高频事件不再反复重建全部地块 DOM。

## 二、鼠标右键返回

非文本编辑状态下，鼠标右键按以下优先级返回：

1. 关闭桌面更新或桌面备份窗口；
2. 关闭当前编辑弹窗；
3. 关闭地块对比、孤立对象检查、面积编辑、Markdown 导入或博物志工作台；
4. 收起精细地图的同坐标对象列表；
5. 关闭地块翻面；
6. 退出空间聚焦；
7. 清空当前搜索。

在输入框、文本框和可编辑区域中，右键仍保留系统复制、粘贴菜单。

## 三、一键发布

从 GitHub 克隆仓库后，在根目录双击 `发布新版本.cmd`。

脚本不会把 GitHub Token 或 Tauri 更新私钥写入程序。它只调用本机 Git 登录凭据推送代码和标签；GitHub Actions 从仓库 Secrets 读取签名私钥。

发布前脚本会检查：

- Git、Node.js、npm 是否可用；
- 当前目录是否为 Git 仓库；
- `origin` 是否为 `a58293/SHmap`；
- 新版本是否低于当前版本；
- 本地或远程是否已经存在同名标签；
- 待提交文件中是否疑似包含 `.env`、`.key` 或签名私钥；
- 项目校验和前端构建是否成功。

确认后依次执行：版本同步、依赖安装、校验、构建、Git 提交、标签创建、分支推送和标签推送。标签格式固定为：

```text
desktop-v0.3.0
```

GitHub 收到标签后自动运行 `Publish Windows Desktop Update`，生成正式 Release、NSIS 安装程序、签名文件和 `latest.json`。
