# SHmap｜山海经原典地图研究台

基于《山海经》原文、古注、平行古籍和项目推导建立的可持续扩充地图研究工具。

## 当前版本

- 程序：`Demo v037`
- 地图数据：`v031-r0001`
- 研究方法：`v002`
- 制图执行规则：`v004`


## v037 主要调整

- 拖动地图时复用已有地块DOM，仅更新摄像机与Canvas；松手后再完整重绘。
- 连续滚轮缩放时使用轻量图层变换，停止后统一刷新。
- 对象、主格、区域、搜索、空间关系和孤立审计建立缓存。
- 小地图底图只在数据或尺寸改变时重画，移动时只更新视野框。
- 画笔笔迹独立刷新并在落笔后抽稀，不再随每个采集点重建整张地图。
- 本地工作区采用合并保存；高倍率标签避让延迟到视野稳定后执行。
- 保留红色画笔、地图总览跳转、区域概览和高倍率悬停关闭等既有功能。

## 直接运行

进入 [`app`](app/) 文件夹，双击 `启动山海经地图Demo.bat` 或打开 `index.html`。

也可以直接打开根目录的 `index.html`，它会跳转到程序入口。

## 当前地图规则

- 都广之野中心固定为 `(0, 0)`；
- 底层坐标单位为“里”；
- 1个主格 = 100里；
- 格内按10里精度表达，底层可保留更精确坐标；
- 同一地块和同一精确坐标均允许多个对象共存；
- 普通河流使用线型表达，海、泽、湖、渊、池等水体才使用面积；
- 面积、作用域、点对象和文本归属必须分层判断。

## v014 本地编辑轮次

- “仅导出更改包”：下载 `.shjpatch`，但继续保留本轮更改。
- “完成本轮编辑”：下载后可归档本轮，未提交计数归零。
- 历史提交可在“更改记录”中查看，并手动标记为已上传 GitHub。
- 回收站支持恢复、永久删除、清空以及保留期限设置。

## GitHub pending 应用

- GitHub Desktop 将更改包上传到 `submissions/pending/`。
- 程序点击“检查数据更新”即可发现新包。
- 点击“查看并应用”会先下载、解析并显示每一项变化。
- 有冲突时不会写入；无冲突时一次性保存到本地地图。
- 同一台电脑已经包含的变化会显示为“已包含”，可直接标记同步。


## v025 高倍率彩色精细地图

- 放大到420%以上自动进入10里彩色精细地图，缩小到380%以下返回100里地块卡片。
- 地形主要用颜色、连续色块与透明度表达；对象名称和坐标作为辅助信息。
- 对象按真实里坐标显示；同一精确坐标的多个对象可展开选择。
- 仅对已录入路径节点的河流绘制连续线路和流向箭头。
- 未录入路径或明确边界的数据不会被程序强行连接。


## v036 主要功能

- 地图放大至800%以上时，自动关闭悬停透镜，避免遮挡精细地图。
- 缩小至650%以下后恢复悬停透镜。
- 跨越阈值时立即关闭已经显示的弹窗，并使用缓冲区避免反复闪烁。


## v037 性能优化

- 地图对象、主格、精确坐标组、面积与线对象建立一次性索引。
- 搜索、空间聚焦、区域概览和孤立对象审计按数据版本缓存。
- 拖动地图时复用现有地块DOM，仅平移图层并刷新Canvas。
- 滚轮缩放在操作期间使用轻量变换，停止后再完整重绘。
- 小地图底图仅在数据或尺寸变化时重画，普通移动只更新视野框。
- 画笔笔迹独立按动画帧刷新，划过地块不再触发整张DOM重建。
- 本地工作区改为450毫秒合并保存，页面隐藏或关闭前强制写入。
- 高倍率标签避让改为停止移动后延迟执行。
