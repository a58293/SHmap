# 山海经原典地图研究台 · 桌面版 v002 客户端更新交付说明

## 这版增加了什么

- 程序右上角增加“检查更新”。
- 显示当前版本、可用版本、版本说明和下载进度。
- 每24小时自动检查一次，可在更新窗口中关闭。
- 安装前强制保存SQLite工作区，并创建“更新前备份”。
- 只安装使用项目固定私钥签名的更新包。
- 正式版本由GitHub Actions发布到GitHub Releases，并生成客户端读取的 `latest.json`。

## 一、先覆盖仓库

1. 解压“SHmap_仓库更新包_桌面版_v002_客户端更新_可直接上传.zip”。
2. 将内部文件覆盖到本地 `SHmap` 仓库根目录。
3. 在GitHub Desktop提交：`增加客户端签名更新功能 v002`。
4. 点击 `Push origin`。

## 二、设置更新签名密钥

更新密钥在单独的“仅本地保管”压缩包中，绝不能放进SHmap仓库。

1. 登录GitHub并进入 `a58293/SHmap`。
2. 打开 `Settings`。
3. 进入 `Secrets and variables` → `Actions`。
4. 点击 `New repository secret`。
5. 新建 `TAURI_SIGNING_PRIVATE_KEY`，粘贴密钥包中 `TAURI_SIGNING_PRIVATE_KEY.txt` 的全部内容。
6. 再新建 `TAURI_SIGNING_PRIVATE_KEY_PASSWORD`，粘贴密码文件的全部内容。
7. 两个Secret都保存后，不要把密钥文件上传到GitHub、公开网盘或群聊。

## 三、第一次生成v002安装包

1. 打开仓库的 `Actions`。
2. 运行 `Build Windows Desktop`。
3. 变成绿色对勾后，在Summary底部下载 `山海经原典地图研究台-Windows-v002`。
4. 解压并安装其中的NSIS安装程序。

v001没有内置更新器，因此必须手动安装v002一次。从v002开始，后续版本才能在客户端内更新。

## 四、以后发布v003、v004

1. 修改程序版本号和版本说明。
2. 推送代码。
3. 在Actions运行 `Publish Windows Desktop Update`。
4. 工作流会签名安装包、创建GitHub Release并生成 `latest.json`。
5. v002及以上客户端点击“检查更新”即可下载安装。

同一版本号不要重复运行正式发布流程，否则会与已有Git标签或Release冲突。

## 五、两个工作流的区别

- `Build Windows Desktop`：测试编译和下载安装包，不发布客户端更新。
- `Publish Windows Desktop Update`：正式签名并发布，客户端会读取到它。

## 六、数据安全

程序更新的是安装目录，不替换用户数据目录中的 `shmap.db` 和 `backups`。客户端在安装更新前还会额外建立一份更新前备份。
