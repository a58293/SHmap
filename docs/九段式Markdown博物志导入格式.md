# 九段式 Markdown 博物志导入格式（v0.4.3）

文件名必须与客户端中需要更新的对象名称一致，例如：

```text
九疑山.md
```

程序通过文件名匹配既有对象。导入仅更新博物志，不新建对象、不改变坐标。

```markdown
## 01.区域定位

区域定位内容

## 02.一句话概要

一句话概要

## 03.本地标签

#标签一 / #标签二

## 04.地块属性

1，地貌类型：
2，水文特征：
3，方位范围：
4，典籍出处：

## 05.数据关系

1，父级区域：
2，相邻地块：
3，相关水域：
4，相关生灵：

## 06,分别整理以下六类：

### 地貌

条目名称

1，与本地关系：
2，核心特征：
3，证据：

### 水系

原文未载明确

### 草木

原文未载明确

### 鸟兽

原文未载明确

### 金玉矿物

原文未载明确

### 人群神祇

原文未载明确

### 事件与遗迹

原文未载明确

## 07,原文摘录

原文摘录

## 08，其他典故

古注或其他典籍补充

## 09，详细描述

完整描述
```

说明：

- 标题可使用中文或英文逗号、顿号、句点作为编号后的分隔符。
- 第06节中，“原文未载明确”不会生成卡片。
- 条目可增加“4，功效：”。
- 无法唯一匹配客户端对象的条目仍会保留为博物志文字卡片。
- 原有 `### 对象：` 和 `### 地块档案：` 格式继续支持。
