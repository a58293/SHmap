# SHmap 纯地图调研 Markdown 导入规范
## 统一格式（不含任何玩法或扩展设定字段）

> 以后导入地图资料，统一使用本文件中的格式。  
> 一个 Markdown 文件可以同时包含多个“地块档案”和多个“对象”。  
> 只填写本次需要新增或修改的字段；未改动的字段直接删除，不要保留空行。

---

# 一、通用规则

## 1. 地块档案标题

必须使用三个 `#`：

```md
### 地块档案：X+001_Y+014
```

这里的坐标是 **100里主格编号**。

## 2. 对象标题

必须使用三个 `#`：

```md
### 对象：对象名称
```

## 3. 字段格式

统一使用：

```md
* 字段名：字段内容
```

多行内容写法：

```md
* 详细：
  第一段内容。
  第二段内容。
```

## 4. 更新规则

- 没写出的字段：保持原内容不变。
- 写出字段并填写内容：新增或覆盖该字段。
- 写出字段但冒号后留空：清空原字段。
- 同一个文件可以包含多个地块和多个对象。
- 已有对象必须继续使用原对象ID。
- 新对象必须使用新的唯一对象ID。

## 5. 不确定的位置

精确坐标不确定时，不要编造坐标。使用：

```md
* 主格坐标：X+001_Y+014
* 坐标性质：同格待定
```

精确坐标明确时，使用：

```md
* X坐标（里）：160
* Y坐标（里）：1440
```

## 6. 重叠对象

重叠对象不需要填写“数量”或“重叠关系”字段。

多个对象精确坐标完全相同时，程序自动显示为：

```text
玄鸟 / 玄蛇 / 玄豹
```

并生成重叠卡片翻转功能。

## 路径节点规则

- 至少两个节点才能绘制线条。
- 每个节点格式必须是 `X,Y`。
- 节点顺序就是显示顺序。
- 第一个节点为起点。
- 最后一个节点为终点。
- 不确定河段不要补造节点。

## 路径状态建议值

```text
原文明载
部分明确
古注支持
现代考证
候选
待核对
```

## 水系与地貌关系建议值

```text

在此汇入
形成湖／泽／池／渊
仅确定同格，路径未定
```
